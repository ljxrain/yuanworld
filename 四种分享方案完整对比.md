## 方案1：口令分享 📋

### 基本信息

- **图标**：📋
- **名称**：口令分享
- **标签**：微信推荐、100%防封
- **成本**：¥0

### 工作原理

```
用户A点击"口令分享"
    ↓
打开口令页面，显示：
🎨 #博世界AI8888#
复制这段话，打开手机浏览器访问 http://49.232.220.223/yuan
输入邀请码即可领取AI形象照优惠，我们都能获得推荐奖励！💰
    ↓
用户A一键复制 → 发送到微信/QQ
    ↓
用户B复制文字 → 打开浏览器 → 访问网站
    ↓
网站自动检测剪贴板 → 弹窗"检测到邀请码"
    ↓
点击"立即使用" → 自动绑定邀请关系
```

### 技术实现

**前端页面：**
- `invite-code-share.html` - 口令生成和显示
- `clipboard-detector.js` - 自动检测剪贴板

**口令格式：**
```
🎨 #博世界AI{邀请码}#
复制这段话，打开手机浏览器访问...
```

**剪贴板检测：**
```javascript
const pattern = /#博世界AI([A-Z0-9]{8})#/;
const text = await navigator.clipboard.readText();
const match = text.match(pattern);

if (match) {
    showInviteModal(match[1]); // 显示邀请弹窗
}
```

### 优势

✅ **100%防封** - 纯文本，微信无法检测
✅ **零成本** - 不需要域名、服务器等额外开销
✅ **用户习惯** - 淘宝、抖音已教育市场，用户熟悉
✅ **自动化高** - 剪贴板自动检测，用户无需手动输入

### 劣势

❌ 需要用户手动复制粘贴（多一步操作）
❌ 视觉冲击力不如海报
❌ Safari浏览器可能拒绝剪贴板权限

### 适用场景

**最佳场景：**
- 微信群聊
- 微信朋友圈（文字分享）
- QQ群
- 短信

**不适合：**
- 线下推广（无海报展示）
- 需要视觉冲击的场景

### 使用步骤

1. 分销中心 → 点击"📋 口令分享"
2. 一键复制口令
3. 发送到微信/QQ等聊天工具
4. 好友复制后打开浏览器访问网站
5. 系统自动检测并绑定

## 方案2：活码海报 🔥

### 基本信息

- **图标**：🔥
- **名称**：活码海报
- **标签**：可切换域名、抗封
- **成本**：¥2600/年（52个域名）

### 工作原理

```
用户A点击"活码海报"
    ↓
后台调用API创建活码
POST /api/qrcode/create
    ↓
生成短链接：http://49.232.220.223/q/QA1B2C3D
    ↓
生成二维码海报（短链接嵌入）
    ↓
用户A保存海报 → 发到朋友圈/线下展示
    ↓
用户B扫码 → 访问 /q/QA1B2C3D
    ↓
数据库查询当前可用域名
    ↓
301跳转到真实页面，绑定邀请关系

--- 如果域名被封 ---
管理员后台切换域名
所有活码自动使用新域名
用户手中的海报图片无需重新生成！
```

### 技术实现

**数据库表：**
```sql
domain_pool          -- 域名池
dynamic_qrcodes      -- 活码表
qrcode_scan_logs     -- 扫码日志
```

**短链接系统：**
```javascript
// 访问 /q/QA1B2C3D
router.get('/q/:code', async (req, res) => {
    // 查询活码
    const qrcode = await getQRCode(req.params.code);

    // 查询当前可用域名
    const domain = await getCurrentDomain(qrcode.current_domain_id);

    // 跳转
    res.redirect(302, qrcode.target_url);
});
```

**域名切换：**
```javascript
// 域名被封后，批量切换
POST /api/qrcode/batch-switch
{
    oldDomainId: 1,  // 被封的域名
    newDomainId: 5   // 新域名
}
```

### 优势

✅ **抗封能力强** - 被封后可后台切换域名
✅ **海报永久有效** - 图片不变，后台切换跳转地址
✅ **用户体验好** - 扫码即可，无需手动操作
✅ **数据统计** - 可查看扫码次数、转化率等

### 劣势

❌ 需要域名池（有成本）
❌ 依然有被封风险（虽然可切换）
❌ 需要管理员手动切换域名

### 适用场景

**最佳场景：**
- 线下推广（门店展示、展会、传单）
- 大规模推广（印刷物料多）
- 朋友圈海报分享
- 公众号推文

**不适合：**
- 纯线上测试（成本高）
- 小规模推广（准备域名麻烦）

### 成本分析

**域名池方案：**
```
每个域名成本：50元/年
准备数量：52个（一年用量，假设1个/周被封）
年成本：52 × 50 = 2600元
```

**对比不使用活码：**
```
每次被封后重新设计海报：500元
重新印刷物料：2000元
一年被封10次：(500 + 2000) × 10 = 25000元

节省成本：25000 - 2600 = 22400元（节省90%）
```

### 使用步骤

1. 分销中心 → 点击"🔥 活码海报"
2. 系统生成短链接和二维码
3. 保存海报图片
4. 发送到朋友圈或用于线下展示
5. 用户扫码自动跳转并绑定

**域名被封后：**
1. 后台标记域名已封：`POST /api/qrcode/mark-blocked`
2. 添加新域名：`POST /api/qrcode/add-domain`
3. 批量切换：`POST /api/qrcode/batch-switch`
4. 所有活码自动使用新域名

## 方案3：普通海报 🖼️

### 基本信息

- **图标**：🖼️
- **名称**：普通海报
- **标签**：口令二维码、100%防封
- **成本**：¥0

### 工作原理

```
用户A点击"普通海报"
    ↓
生成二维码（内容是口令文本，不是链接！）
二维码内容：
🎨 #博世界AI8888#
复制这段话，打开手机浏览器访问...
    ↓
生成精美海报
    ↓
用户A保存海报 → 发到朋友圈
    ↓
用户B扫码 → 看到一段文字（不是跳转网页）
    ↓
用户B长按文字 → 复制
    ↓
用户B打开浏览器 → 访问网站
    ↓
网站自动检测剪贴板 → 绑定邀请关系
```

### 技术实现

**关键代码：**
```javascript
// qrcode-modal.html 第235-237行
const commandText = `🎨 #博世界AI${invitationCode}#
复制这段话，打开手机浏览器访问 http://49.232.220.223/yuan
输入邀请码即可领取AI形象照优惠，我们都能获得推荐奖励！💰`;

// 二维码内容是文本，不是URL
QRCode.toCanvas(canvas, commandText, {
    width: 200,
    height: 200
});
```

### 为什么不会被封？

**普通二维码（会被封）：**
```
二维码内容 = http://abc.com/register?code=8888
微信扫码 → 检测到URL → 可能拦截
```

**口令二维码（不会被封）：**
```
二维码内容 = 🎨 #博世界AI8888#\n复制这段话...
微信扫码 → 显示文字内容 → 无法拦截（不是URL）
```

### 优势

✅ **100%防封** - 二维码内容是文本，不是链接
✅ **零成本** - 不需要域名
✅ **视觉效果好** - 精美海报，有吸引力
✅ **用户教育成本低** - 类似淘宝口令

### 劣势

❌ 用户需要扫码后手动复制（多一步操作）
❌ 相比活码，用户体验稍差一点

### 适用场景

**最佳场景：**
- 微信朋友圈海报分享
- 公众号文章配图
- 微信群海报

**不适合：**
- 线下快速推广（用户嫌麻烦）
- 需要即刻转化的场景

### 使用步骤

1. 分销中心 → 点击"🖼️ 普通海报"
2. 系统生成口令二维码海报
3. 保存海报图片
4. 发送到朋友圈/微信群
5. 用户扫码 → 看到文字 → 复制 → 打开浏览器 → 自动绑定

```javascript
// 页面加载时检测URL参数
```

### 优势

## ⚠️ 常见问题

### Q1: 哪个方案防封效果最好？

**A:** 📋 口令分享 和 🖼️ 普通海报（口令二维码）

**原因：**
- 内容是纯文本，不是链接
- 微信无法检测和拦截
- 淘宝、抖音同样原理，使用多年未被封

### Q2: 直接链接在微信能用吗？

**A:** 不能！微信会拦截外部链接。

**表现：**
- 点击链接 → 提示"已停止访问该网页"
- 或者需要"在浏览器中打开"

**解决方案：**
- 微信内使用📋 口令分享
- 微信外（QQ/短信）使用🔗 直接链接

### Q3: 活码海报需要准备多少个域名？

**A:** 取决于推广力度。

**参考：**
```
小规模（日扫码 < 100）：
- 主站IP：1个
- 备用域名：5个
- 预计被封速度：1个/月

中等规模（日扫码 100-1000）：
- 主站IP：1个
- 备用域名：20个
- 预计被封速度：1个/周

大规模（日扫码 > 1000）：
- 主站IP：1个
- 备用域名：50+个
- 预计被封速度：1个/天
```

### Q4: 可以同时使用多个方案吗？

**A:** 可以！而且推荐这么做。

**最佳组合：**
```
对不同用户使用不同方案：
- 微信用户 → 口令分享/普通海报
- QQ用户 → 直接链接
- 线下用户 → 活码海报

对同一个用户提供多个选择：
"您可以复制口令，也可以扫码，还可以点击链接"
```

### Q5: 四个方案哪个转化率最高？

**A:** 🔗 直接链接（点击即可，最简单）

**转化率排名：**
```
1. 🔗 直接链接（90%+） - 点击即可
2. 🔥 活码海报（70%+） - 扫码即可
3. 🖼️ 普通海报（50%+） - 扫码+复制
4. 📋 口令分享（40%+） - 复制+打开浏览器
```

**但注意：**
- 转化率高不代表适合所有场景
- 微信场景必须用防封方案
- 要综合考虑防封+转化率

```
- 口令复制率
```

**方案4：直接链接**
```
- 链接点击次数
- 来源分析（QQ/短信/邮件）
- 直接转化率
```

## 🎉 总结

### 核心要点

1. **没有完美方案，只有最合适的方案**
2. **微信场景必须防封（口令/口令二维码）**
3. **非微信场景直接链接最好用**
4. **组合使用效果最佳**

### 推荐策略

**个人推广（小规模）：**
```
- 主力：📋 口令分享（零成本，防封）
- 备用：🔗 直接链接（QQ/短信用）
```

**团队推广（中规模）：**
```
- 微信：📋 口令分享 + 🖼️ 普通海报
- 其他：🔗 直接链接
```

**公司推广（大规模）：**
```
- 线下：🔥 活码海报（印刷物料）
- 线上：📋 口令分享（微信）+ 🔗 直接链接（其他）
```
